Build a Python MVP for an agentic AI surveillance system. Use a feature-first file structure. Do not put everything in one file. Here is the exact structure to follow:

surveillance-mvp/
├── main.py                         # entry point, runs the full pipeline loop
├── config.py                       # ALL hardcoded values live here, nothing else
├── features/
│   ├── ingestion/
│   │   └── frame_sampler.py        # OpenCV reads MP4, yields frames every 3 seconds
│   ├── detection/
│   │   └── vlm_detector.py         # sends frame to GPT-4o Vision, returns structured threat JSON
│   ├── risk/
│   │   └── risk_scorer.py          # LLM-backed risk scoring, returns risk score + escalation mode
│   ├── agents/
│   │   ├── graph.py                # LangGraph graph definition, nodes and edges wired here
│   │   ├── context_enricher.py     # attaches camera profile + time of day to state
│   │   ├── escalation_agent.py     # LLM call, generates incident summary + recommended action
│   │   └── dispatch_agent.py       # updates dashboard state, handles HITL interrupt for mode 3
│   ├── audit/
│   │   └── audit_logger.py         # appends every incident to in-memory list, assigns case ID
│   └── dashboard/
│       └── app.py                  # Streamlit UI, reads shared state, renders everything
└── requirements.txt

---

RULES YOU MUST FOLLOW:
- Every file does ONE thing only
- config.py is the only place with hardcoded values. Every other file imports from config.py
- No file should exceed 80 lines. If it does, split it
- Use type hints everywhere
- Each function has a one-line docstring
- State is a single TypedDict called IncidentState defined in agents/graph.py and imported everywhere it is needed

---

config.py must contain:
- CAMERA_PROFILE dict: camera_id, location_name, zone_type (transit), coordinates
- CROWD_DENSITY_RULES: dict mapping hour ranges to "low" / "medium" / "high"
- ESCALATION_RULES: maps threat_type + crowd_density to mode 1, 2, or 3
- OPENAI_API_KEY loaded from .env
- VIDEO_PATH: path to the demo MP4 file
- FRAME_INTERVAL_SECONDS: 3

---

IncidentState TypedDict must have these fields:
- case_id: str
- camera_profile: dict
- timestamp: str
- frame_b64: str
- frame_description: str
- threat_detected: bool
- threat_type: str           # "weapon" | "physical_altercation" | "suspicious_behaviour" | "none"
- confidence: str            # "high" | "medium" | "low"
- people_count: int
- crowd_density: str
- risk_score: float
- escalation_mode: int       # 1, 2, or 3
- incident_summary: str
- recommended_action: str
- human_approved: bool
- dispatch_status: str       # "pending" | "awaiting_confirmation" | "dispatched"
- audit_trail: list[str]

---

vlm_detector.py must:
- Accept a base64 encoded frame
- Call GPT-4o Vision (model: gpt-4o) using the openai python SDK
- Use a system prompt that returns structured detection JSON and may include a backup LLM risk score, but does not set the final authoritative score
- Parse the JSON response and return it as a dict
- Handle JSON parse errors gracefully, return threat_detected: false on failure

---

risk_scorer.py must:
- Use an LLM to assign the final risk_score as a float from 0.0 to 10.0
- Use the frame plus detection context as inputs
- Map the LLM score into color and escalation mode after scoring
- Fall back safely if the risk-model response is unavailable

---

LangGraph graph in graph.py must have these nodes in order:
1. context_enricher
2. vlm_detect (calls vlm_detector.py)
3. risk_score (calls risk_scorer.py)
4. escalation_agent (LLM call, generates incident_summary and recommended_action)
5. dispatch_agent (conditional: if mode 3, interrupt_before this node for HITL)
6. audit_logger

Use interrupt_before=["dispatch_agent"] on the graph compile when escalation_mode == 3.
Use a MemorySaver checkpointer.

---

Streamlit dashboard in app.py must:
- Display in two columns
- Left column: current frame image, camera name, location, timestamp, crowd density
- Right column: alert mode badge (green for no threat, amber for mode 1-2, red for mode 3), incident_summary, recommended_action, HITL confirm button (only visible when dispatch_status == "awaiting_confirmation"), scrollable audit trail log
- Use st.session_state to share IncidentState between pipeline and UI
- Refresh every 3 seconds using st.rerun()
- Alert badge must use st.markdown with inline HTML for color: green / orange / red

---

main.py must:
- Initialize Streamlit session state
- Start frame_sampler loop
- For each frame, run the LangGraph graph with thread_id = case_id
- If escalation_mode == 3 and human_approved == False, pause and set dispatch_status = "awaiting_confirmation"
- When HITL confirm button is clicked, resume graph with human_approved = True
- Pass updated state to audit_logger after every run

---

requirements.txt must include:
openai
langgraph
langchain-openai
opencv-python
streamlit
python-dotenv
pillow
